# 🚀 Week 5 Big Data Assignment – PySpark

This project demonstrates **Spark fundamentals** and hands‑on practice with **data cleaning, transformation, and aggregation** using PySpark DataFrames.  
It covers key concepts like MapReduce limitations, Spark’s in‑memory processing, DataFrame immutability, filtering, grouping, and building a complete processing pipeline.

---

## 📂 Project Structure
- **Q1–Q2** → Spark vs MapReduce fundamentals  
- **Q3–Q4** → Removing duplicates & filtering with groupBy  
- **Q5–Q6** → Handling nulls and conditional aggregations  
- **Q7–Q8** → DataFrame immutability & filtering with conditions  
- **Q9–Q11** → Null handling, shuffle process, wide transformations  
- **Q12–Q14** → Cleaning null/empty values, multiple aggregations, schema risks  
- **Q15** → End‑to‑end pipeline combining cleaning + aggregation  

---

## 🛠️ Technologies Used
- **Python 3.12**
- **PySpark 4.0.3**
- **Google Colab / Jupyter Notebook**

---

## ✨ Key Highlights
- ⚡ **In‑Memory Computing** → Faster iterative ML compared to MapReduce  
- 🧹 **Data Cleaning** → Remove duplicates, handle nulls, drop/rename columns  
- 🔎 **Filtering** → Age ranges, subscription types, regions, categories  
- 📊 **Aggregation** → Count, sum, avg, min, max with `.agg()`  
- 🔄 **Shuffle Process** → Understanding wide transformations in Spark  
- 🏗️ **Pipeline** → Deduplication → Null handling → GroupBy → Revenue calculation
